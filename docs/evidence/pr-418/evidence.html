PR #418: Human-in-the-Loop Discussion Before Compilation

Evidence report for issue #418 -- interactive compilation review workflow

What was built

An optional "discuss before compile" mode that lets users review key takeaways, provide focus guidance, and approve or reject compilation drafts before articles are saved to the wiki.

  • New REVIEW_PENDING ingest status -- sources pause in this state until the user reviews
  • CompilationDraft model -- persists the LLM's draft output, extracted takeaways, and user guidance
  • Takeaway extraction -- a separate LLM call extracts 3-10 key findings before full compilation
  • Guided recompilation -- when the user provides focus guidance, the source is re-compiled with their direction
  • Draft API endpoints -- GET/POST for reviewing, approving, and rejecting drafts
  • DraftReviewPanel UI -- inline review widget with takeaway list, draft preview, and guidance input
  • WebSocket event -- draft.ready notifies the frontend when a draft is ready for review
  • Opt-in via config -- WIKIMIND_COMPILER__INTERACTIVE=true (default: false)

1. Inbox View with Review-Pending Sources

Sources in review_pending state show an amber Review badge and a "Review draft" button.

2. Draft Review Panel

Clicking "Review draft" opens the DraftReviewPanel inline. It shows the title, summary, 5 numbered key takeaways extracted by the LLM, and action buttons.

3. Draft Article Body Preview

Users can expand the full draft article body to review what the LLM generated before approving.

4. Guided Compilation

When the user types focus guidance, the "Approve as-is" button changes to "Approve with guidance". The source will be re-compiled with the user's direction weighted into the prompt.

API Endpoints Added

GET /api/ingest/sources/{id}/draft

Returns the pending compilation draft with key takeaways and draft article body.

POST /api/ingest/sources/{id}/draft/approve

Approves the draft, optionally with guidance for re-compilation.

POST /api/ingest/sources/{id}/draft/reject

Rejects the draft and resets the source to pending.
