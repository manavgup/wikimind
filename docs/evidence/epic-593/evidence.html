#540: semantic-release push blocked by branch ruleset (GH013)

Diagnosis: Investigated the branch ruleset via GitHub API:

$ gh api repos/manavgup/wikimind/rulesets/14794262
rules: [deletion, non_fast_forward, pull_request (0 approvals)]
bypass_actors: [RepositoryRole:5 (Admin), bypass_mode: always]

The manavgup account IS admin and admins ARE in the bypass list. The SEMANTIC_RELEASE_GITHUB_TOKEN secret exists and the url.insteadOf git config IS applied (confirmed in workflow logs). Yet the push is rejected with GH013.

Root cause: The token is a fine-grained PAT (github_pat_...). Fine-grained tokens cannot bypass repository rulesets — this is a known GitHub limitation. Even though the user is admin, the fine-grained token doesn't inherit ruleset bypass.

Fix applied: Replaced the fine-grained PAT with a classic PAT with repo scope. Classic tokens authenticate as the user with full role-based permissions, including ruleset bypass.

$ gh secret set SEMANTIC_RELEASE_GITHUB_TOKEN
Set Actions secret SEMANTIC_RELEASE_GITHUB_TOKEN for manavgup/wikimind
