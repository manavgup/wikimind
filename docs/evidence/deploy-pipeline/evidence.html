4. Known Issue: Missing compiled_at Column

UndefinedColumnError on /api/wiki/articles

The article.compiled_at column exists in the SQLModel definition (models.py:209) but has no Alembic migration. It was added during development against SQLite (where create_all() handles it), but production Postgres only runs Alembic migrations.

Fix: Create Alembic migration 0012_add_article_compiled_at.py with ALTER TABLE article ADD COLUMN compiled_at TIMESTAMP.

40 PRs merged in 4 days. 60% were deploy/infra fixes. Analysis of why.

Fix-the-Fix Chains

Redis Connectivity Saga (9 PRs, 3 days)

#561 → #566 → #572 → #575 → #577 → #579 → #583 → #611 → #612

Root cause: Nobody ran flyctl redis create or tested Redis connectivity manually before writing CI. Every assumption about CLI flags, output format, and TLS requirements was guessed from docs and proved wrong. PRs #583, #611, #612 each contradicted the previous one's TLS assumption within 12 hours.

PgBouncer / Postgres Surprise (4 PRs, 1 day)

#583 → #606 → #618 → #622

Root cause: Did not know Fly.io Postgres uses PgBouncer in transaction-pooling mode. This is the #1 documented gotcha in Fly Postgres docs. statement_cache_size=0 and idle connection handling should have been configured from day one.

Machine Lifecycle Misunderstanding (3 PRs, 1 day)

#580 → #601 → #618

Root cause: Fly.io auto-stops machines with no HTTP traffic. Workers don't serve HTTP. Smoke tests assumed machines stayed running between requests. All documented in Fly machine lifecycle docs.

Smoke Test / Token Bootstrapping (3 PRs, 2 days)

#563 → #573 → #623

Root cause: Smoke tests built incrementally in response to failures. The auth token format mismatch (dev_token vs JWT) was written without understanding the auth flow.

6. Recommendations

R1: Pre-Automation Checklist
Before writing CI for a new Fly.io service, manually verify from inside a VM: Redis connectivity (redis:// vs rediss://), PgBouncer behavior (statement_cache_size), machine lifecycle (auto-stop), flyctl CLI flags (--help first).
R2: Docker Compose Fly Parity
Add docker-compose.fly-parity.yml with PgBouncer in transaction-pooling mode, Redis without TLS, multiple Gunicorn workers. Run in CI before deploy.
R3: One Infra Change Per PR
Each infrastructure PR must include manual verification evidence (terminal output from flyctl ssh console). No "should work" PRs.
R4: Static Secrets, No CLI Scraping
Provision Redis/Postgres once manually, store URLs as GitHub secrets. Deploy workflow verifies connectivity only, never discovers or creates resources.
R5: Separate Provisioning from Deploy
Split into infra-provision.yml (manual, rare) and deploy.yml (automatic, every push). Deploy assumes infra exists.
R6: Fly-Specific Integration Tests
Add tests for: asyncpg + PgBouncer compatibility, concurrent init_db(), long-running queries with idle connections. Run against local Postgres+PgBouncer in CI.
