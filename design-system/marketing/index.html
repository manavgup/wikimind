the gap

RAG re‑discovers. WikiMind remembers.

Most LLM‑plus‑documents tools are a retrieval layer over raw files. Every question starts from scratch. Nothing accumulates. WikiMind flips the arrow: compile once, keep it current, and the LLM reads the wiki instead of the documents.

retrieval · the status quo

Knowledge that evaporates between queries.

  • Ephemeral. Every question re‑retrieves chunks from raw docs. Nothing is kept.
  • Flat. Cross‑references are re‑discovered on the fly — often imperfectly.
  • Opaque. You can't browse "what the LLM knows" — only watch it answer.
  • No synthesis. Questions spanning five documents get re‑solved every time.

compilation · WikiMind

A compounding artifact you can actually read.

  • Persistent. Every source is compiled into markdown pages you can browse.
  • Linked. Cross‑references are already there. Backlinks, graph view, concept trees.
  • Inspectable. Every claim carries a confidence tag: sourced · mixed · inferred · opinion.
  • Cumulative. Synthesis happens once at compile‑time, then gets revised — not re‑derived.
three operations

Feed. Ask. Lint.

Everything you do with WikiMind falls into one of three verbs. The first adds to the wiki. The second queries it. The third keeps it healthy.

01 / INGEST

Feed

Drop a URL, PDF, YouTube link, or podcast into the Inbox. The LLM reads it, discusses key takeaways with you, writes a summary page, and updates 10–15 existing pages where it matters.

the three layers · in detail

Separate what the LLM owns from what you own.

Clear boundaries are the whole trick. Your sources are immutable. The wiki is LLM‑authored and versioned in git. The schema is a markdown file you edit together.

Layer 01

Raw sources

Your curated collection — articles, papers, transcripts, images. The LLM reads from it, never modifies it. This is your source of truth.

raw/ ├── articles/ ├── pdfs/ ├── transcripts/ └── assets/
Owner: you
Layer 02

The wiki

A directory of generated markdown. Entity pages, concept pages, comparisons, an index, a log. The LLM creates and maintains it as new sources arrive.

wiki/ ├── index.md // catalog ├── log.md // timeline ├── concepts/ └── entities/
Owner: the LLM
Layer 03

The schema

A single configuration file — CLAUDE.md or AGENTS.md. Describes conventions, workflows, and page formats.

AGENTS.md # how to ingest a source # how to answer a query # when to file back # lint rules
Owner: you and the LLM, together

the lineage

A Memex that maintains itself.

In 1945 Vannevar Bush described the Memex — a personal, curated store of documents linked by associative trails. Private. Actively maintained. With the connections between documents as valuable as the documents themselves.

The part he couldn't solve was who does the maintenance. Wikis answered it with volunteers, and fizzled at personal scale. WikiMind answers it differently: the LLM does it. The cost of upkeep falls to near zero. The wiki stays alive.

→ bush, 1945 · "as we may think" · atlantic monthly

questions

Before you ask.

Where does my data live?
Locally. Raw sources and the wiki are plain files on your disk — a git repo, if you want version history. WikiMind never phones home. LLM calls go to whichever provider you've configured.
Which LLM does it use?
Bring your own. Configure any combination of OpenAI, Anthropic, Google, or a local Ollama model in Settings. You'll see the per‑source cost before and after ingest.
How is this different from NotebookLM or ChatGPT file upload?
Those are retrieval systems. The LLM re‑reads your files on every query and never builds anything persistent. WikiMind compiles your sources into a wiki you can actually browse, edit, and version — and the LLM reads the wiki, not the raw files.
What if I disagree with what the LLM wrote?
Every claim has a confidence tag: sourced · mixed · inferred · opinion. Edits are first‑class — you can revise any page and the LLM will respect your version on the next compile. The wiki is yours.
Does it scale?
To a few thousand sources, comfortably. At small scale the index.md catalog is enough for the LLM to navigate. As the wiki grows, an on‑disk BM25 + vector search kicks in automatically so the LLM always gets the right pages into context.
Is it open source?
Yes. Everything — the compiler, the schema format, the web app — ships as MIT on GitHub. The desktop version is an Electron shell around the same bundle.
